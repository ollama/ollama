Claude Desktop can use Ollama Cloud through Claude's third-party inference mode on macOS and Windows, including Claude Cowork and Claude Code inside the app.

<img
  src="/images/claude-cowork-kimi-k2-6.png"
  alt="Claude Cowork using kimi-k2.6 through Ollama Cloud"
  className="rounded-xl"
/>

## Requirements

- Claude Desktop for macOS or Windows
- An [Ollama API key](https://ollama.com/settings/keys)

Set the key in your shell before launching:

```shell
export OLLAMA_API_KEY=your_api_key
```

## Quick setup

```shell
ollama launch claude-desktop
```

This configures Claude Desktop's third-party profile to use `https://ollama.com`, sets your Ollama API key as the gateway credential, lets Claude Desktop discover available models from Ollama Cloud, and restarts Claude Desktop if needed.

`claude-app` is also supported as an alias for `claude-desktop`.

To bring back the usual Anthropic Claude profile later, run:

```shell
ollama launch claude-desktop --restore
```

On Windows, the launcher writes the Claude Desktop profile settings under `%LOCALAPPDATA%` for both observed profile families, including `Claude-3p` and `Claude Nest-3p`.

## Using Ollama Cloud models

After setup, Claude Desktop discovers your available Ollama Cloud models automatically. For example, `kimi-k2.6` appears inside Claude Cowork.

The same model is also available to Claude Code from inside the Ollama app:

![Claude Code using kimi-k2.6 inside the Ollama app](/images/claude-code-kimi-k2-6.png)

## Configure without launching

```shell
ollama launch claude-desktop --config
```

## Restore normal Claude

Switch Claude Desktop back to its usual profile:

```shell
ollama launch claude-desktop --restore
```

Restore clears the launcher-managed third-party profile selection and gateway fields while preserving the saved Ollama API key for the next setup.

If Claude Desktop is running, use `--yes` to approve the restart automatically:

```shell
ollama launch claude-desktop --restore --yes
```
