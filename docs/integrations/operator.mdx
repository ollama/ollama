[Operator](https://github.com/yernur-dev/operator) is a terminal AI coding agent that writes code, runs tests, and manages git workflows autonomously.

Operator works with [Alem AI](https://llm.alem.ai) models through its OpenAI-compatible API.

## Get started

Install Operator globally:

```shell
npm install -g @yernur/operator
```

Then launch:

```shell
operator
```

## Models

Operator works with [Alem AI](https://llm.alem.ai) models:

| Model | Description |
|---|---|
| `qwen3-6` | Qwen 3.6 |
| `gemma4` | Google Gemma 4 |

Access models at [plus.alem.ai/services](https://plus.alem.ai/services).

## Commands

| Command | Description |
|---|---|
| `/model` | Switch model (qwen3-6, gemma4) |
| `/reasoning` | Adjust thinking depth: off → minimal → low → medium → high → xhigh |
| `/export` | Export session to JSON |
| `/import` | Import and resume a session |
| `/new` | Start a new session |
| `/resume` | Resume an existing session |
| `/quit` | Exit |

## CLI usage

```shell
# Interactive mode
operator

# One-liner
operator "find all TODO in the project and fix them"

# With model selection
operator --model qwen3-6 "write tests for src/auth.ts"

# With API key override
operator --api-key "sk-xxxxx" "who am i"
```

## Browser (Puppeteer)

Operator can control a real browser for QA, E2E testing, and UI verification:

| Action | Example |
|---|---|
| Open a page | `open https://mysite.test and take a screenshot` |
| Screenshot | `take a screenshot of the login page` |
| Click / Form | `login to the site, username admin@test, password 123, click Login` |
| JS check | `open the page and check console.log shows 0 errors` |
| E2E test | `go to the registration form, fill fields, submit and verify redirect to /dashboard` |
| Select element | `select the dropdown and choose "Premium"` |

The browser runs headless — no GUI required, works on VPS.

## Deep research

Operator can perform multi-step web research with cross-verification and confidence scoring:

| Feature | Description |
|---|---|
| Multi-source search | Searches DuckDuckGo and Wikipedia in parallel |
| Cross-verification | Compares claims across multiple sources |
| Confidence scoring | Rates each claim (confirmed / likely / unclear) |
| Structured reports | Generates detailed reports with citations |
| Query planning | Breaks complex topics into focused sub-queries |
| Claim scoring | Evaluates individual claims against evidence |

Usage example:

```text
research "compare qwen3 vs gemma4 architecture and performance"
```

Output includes a structured report with sources, confidence levels, and detailed findings.
