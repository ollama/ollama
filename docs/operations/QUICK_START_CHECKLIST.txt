Common Questions:

Q: "What if I break something?"
A: Git is your friend! Every commit is reversible. If something breaks:
   1. git log --oneline (see your commits)
   2. git revert <commit-hash> (undo that change)
   3. Fix locally, try again

Q: "Should I refactor code logic too?"
A: NO! Only move classes to separate files. Don't change what they do.
   This makes refactoring safe and focused.

Q: "How do I keep APIs stable?"
A: Update __init__.py to re-export from new locations:
   # new_module.py contains the class
   # __init__.py imports and exports it
   # External code doesn't notice anything changed

Q: "What if imports get complicated?"
A: It's OK! That's what __init__.py is for:
   ├─ External imports: use public API from __init__.py
   ├─ Internal imports: use specific module files
   └─ Makes refactoring invisible to users
